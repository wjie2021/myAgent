# Python 数据结构与类型

> 作为 Java 程序员，你已经熟悉面向对象的数据结构。Python 的数据结构更灵活，但也有些"坑"需要了解。

## 基本类型

Python 不需要声明变量类型，解释器自动推断：

```python
# 整数
age = 25                    # int

# 浮点数
price = 9.99                # float

# 字符串
name = "张三"               # str

# 布尔值
is_active = True            # bool（注意大写）

# 空值
result = None               # NoneType（类似 Java 的 null）
```

### 对比 Java

```java
// Java 必须声明类型
int age = 25;
String name = "张三";
boolean isActive = true;
Object result = null;
```

Python 的变量就像一个"标签"，贴在值上面。同一个变量可以贴不同类型的值：

```python
x = 10          # x 是 int
x = "hello"     # x 变成 str 了
x = [1, 2, 3]   # x 变成 list 了
```

Java 不允许这样做（除非用 `Object` 类型）。

## List（列表）

Python 的 List 类似 Java 的 `ArrayList`，但更灵活：

```python
# 创建
fruits = ["苹果", "香蕉", "橙子"]

# 访问（从 0 开始）
fruits[0]        # "苹果"（第一个）
fruits[-1]       # "橙子"（最后一个，负数索引从末尾数）

# 修改
fruits[1] = "葡萄"    # 把第二个元素改成"葡萄"

# 添加
fruits.append("芒果")  # 末尾追加

# 删除
fruits.remove("苹果")  # 删除指定元素
fruits.pop(0)          # 删除指定位置的元素

# 长度
len(fruits)            # 获取元素个数

# 切片（获取子集）
fruits[1:3]            # 第2到第3个（左闭右开）
fruits[:3]             # 前3个
fruits[2:]             # 从第3个开始到末尾
```

### 为什么 List 这么重要？

Agent 开发中，List 随处可见：

```python
# API 返回的消息列表
messages = [
    {"role": "user", "content": "你好"},
    {"role": "assistant", "content": "你好！有什么可以帮你的？"},
    {"role": "user", "content": "查天气"}
]

# 工具列表
tools = ["get_weather", "get_attraction", "calculator"]

# 往 messages 里追加消息
messages.append({"role": "assistant", "content": "Thought: ..."})
```

### 遍历 List

```python
# 基本遍历
for fruit in fruits:
    print(fruit)

# 带索引遍历
for i, fruit in enumerate(fruits):
    print(f"{i}: {fruit}")

# 条件筛选
hot_fruits = [f for f in fruits if "热" in f]  # 列表推导式
```

## Dict（字典）

Agent 开发中**最常用**的数据结构。类似 Java 的 `HashMap`，但语法更简洁：

```python
# 创建
person = {
    "name": "张三",
    "age": 25,
    "city": "北京"
}

# 访问
person["name"]           # "张三"（key 不存在会报错）
person.get("name")       # "张三"（推荐，key 不存在返回 None）
person.get("phone", "无")  # "无"（指定默认值）

# 修改
person["age"] = 26

# 添加
person["email"] = "zhangsan@example.com"

# 删除
del person["city"]

# 判断 key 是否存在
"name" in person         # True
"phone" in person        # False
```

### 为什么 Dict 这么重要？

几乎所有 API 的返回值都是 Dict：

```python
# 天气 API 返回
weather = {
    "city": "北京",
    "temperature": 31,
    "condition": "晴",
    "humidity": 45
}

# Agent 的工具定义
tool = {
    "name": "get_weather",
    "description": "查询城市天气",
    "parameters": {
        "city": {"type": "string", "description": "城市名"}
    }
}

# 访问嵌套值
tool["parameters"]["city"]["type"]  # "string"
```

### 遍历 Dict

```python
# 遍历 key
for key in person:
    print(key)

# 遍历 value
for value in person.values():
    print(value)

# 遍历 key-value 对
for key, value in person.items():
    print(f"{key}: {value}")
```

## 嵌套结构

实际 API 返回值通常是 List 嵌套 Dict：

```python
# 比如天气 API 返回
weather_data = {
    "city": "北京",
    "forecast": [
        {"date": "今天", "weather": "晴", "temp": 31},
        {"date": "明天", "weather": "多云", "temp": 28},
        {"date": "后天", "weather": "小雨", "temp": 25}
    ]
}

# 访问嵌套值
weather_data["city"]                        # "北京"
weather_data["forecast"][0]["weather"]      # "晴"
weather_data["forecast"][1]["temp"]         # 28

# 遍历预报列表
for day in weather_data["forecast"]:
    print(f"{day['date']}: {day['weather']}, {day['temp']}度")
```

## Tuple（元组）

类似 List，但**不可修改**。通常用于函数返回多个值：

```python
# 创建
point = (10, 20)

# 访问
point[0]    # 10
point[1]    # 20

# 不可修改
point[0] = 5   # 报错！Tuple 不允许修改

# 函数返回多个值
def get_location():
    return "北京", 39.9, 116.4   # 实际返回的是 Tuple

city, lat, lon = get_location()   # 解包
```

## Set（集合）

类似 Java 的 `HashSet`，用于去重和集合运算：

```python
# 创建
fruits = {"苹果", "香蕉", "橙子", "苹果"}  # 自动去重
print(fruits)   # {"苹果", "香蕉", "橙子"}

# 添加
fruits.add("葡萄")

# 判断是否存在
"苹果" in fruits    # True

# 集合运算
a = {1, 2, 3}
b = {3, 4, 5}
a & b               # {3}（交集）
a | b               # {1, 2, 3, 4, 5}（并集）
a - b               # {1, 2}（差集）
```

## 字符串操作

Python 的字符串是不可变的，但操作方法很丰富：

```python
s = "Hello, World!"

# 访问
s[0]           # "H"
s[-1]          # "!"

# 切片
s[0:5]         # "Hello"
s[7:]          # "World!"

# 常用方法
s.lower()      # "hello, world!"（转小写）
s.upper()      # "HELLO, WORLD!"（转大写）
s.strip()      # "Hello, World!"（去除首尾空白）
s.split(", ")  # ["Hello", "World!"]（分割）
s.replace("Hello", "Hi")  # "Hi, World!"

# 判断
s.startswith("Hello")   # True
s.endswith("!")          # True
"World" in s             # True
```

### f-string 格式化

Agent 开发中拼接提示词常用：

```python
city = "北京"
temp = 31
weather = "晴"

# 基本用法
msg = f"{city}今天{weather}，{temp}度"
# "北京今天晴，31度"

# 表达式
msg = f"温度是 {temp * 9 // 5 + 32} 华氏度"
# "温度是 87 华氏度"

# 多行
prompt = f"""
你是一个旅行助手。
当前城市：{city}
天气：{weather}
温度：{temp}度
请推荐一个合适的景点。
"""
```

## 类型转换

```python
# 字符串 → 整数
age = int("25")

# 整数 → 字符串
age_str = str(25)

# 字符串 → 浮点数
price = float("9.99")

# 字符串 → 列表
chars = list("hello")   # ['h', 'e', 'l', 'l', 'o']

# 列表 → 字符串
words = ["Hello", "World"]
sentence = " ".join(words)   # "Hello World"
```

## Python vs Java 类型对照

| Python | Java | 说明 |
|--------|------|------|
| `int` | `int` / `Integer` | 整数 |
| `float` | `float` / `Double` | 浮点数 |
| `str` | `String` | 字符串 |
| `bool` | `boolean` | 布尔值 |
| `None` | `null` | 空值 |
| `list` | `ArrayList` | 列表 |
| `dict` | `HashMap` | 字典 |
| `tuple` | 不可变数组 | 元组 |
| `set` | `HashSet` | 集合 |
