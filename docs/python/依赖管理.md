# Python 依赖管理

> 背景：作为 Java 程序员，习惯了 Maven 的 pom.xml 自动管理依赖。Python 的机制不太一样。

## 核心概念

### Java — Maven 自动管理

```xml
<!-- pom.xml 声明依赖，Maven 自动下载 -->
<dependency>
    <groupId>com.openai</groupId>
    <artifactId>openai-java</artifactId>
    <version>1.0.0</version>
</dependency>
```

- 运行 `mvn install` 或 IDE 自动拉取依赖
- 依赖和项目绑定，换机器自动同步

### Python — 手动安装

```bash
# 依赖声明在 requirements.txt 里，但需要手动执行安装
pip install -r requirements.txt
```

- Python **不会自动安装依赖**，需要手动执行命令
- 每次换环境（新机器、新虚拟环境）都要重新安装

## 对比表

| 特性 | Java (Maven) | Python (pip) |
|------|--------------|--------------|
| 依赖文件 | pom.xml | requirements.txt |
| 安装命令 | `mvn install`（或 IDE 自动） | `pip install -r requirements.txt` |
| 是否自动 | IDE 自动处理 | **不自动**，必须手动执行 |
| 版本锁定 | pom.xml 精确指定 | requirements.txt 用 `>=` 或 `==` |
| 虚拟环境 | 不需要（Maven 管理） | 强烈建议用 venv |

## requirements.txt 长什么样

```txt
# 简单的文本文件，每行一个包
anthropic>=0.40.0      # >= 表示最低版本
openai==1.50.0         # == 表示精确版本
requests>=2.31.0
python-dotenv>=1.0.0
```

## 常用命令速查

```bash
# 安装 requirements.txt 中的所有依赖（类似 npm install / mvn install）
pip install -r requirements.txt

# 安装单个包
pip install openai

# 安装指定版本
pip install openai==1.50.0

# 查看已安装的包
pip list

# 卸载包
pip uninstall openai

# 导出当前环境的所有依赖（生成 requirements.txt）
pip freeze > requirements.txt
```

## 虚拟环境（venv）

Python 推荐每个项目用独立的虚拟环境，避免依赖冲突（类似 Maven 的项目隔离，但更彻底）。

```bash
# 创建虚拟环境（只需一次）
python -m venv .venv

# 激活虚拟环境（每次打开终端都要执行）
# Windows:
.venv\Scripts\activate
# Mac/Linux:
source .venv/bin/activate

# 激活后，pip install 的包只装到这个项目里
pip install -r requirements.txt

# 退出虚拟环境
deactivate
```

### 为什么需要虚拟环境？

- 项目 A 需要 openai v1.0，项目 B 需要 openai v2.0
- 不用虚拟环境，全局只能装一个版本，会冲突
- 用了虚拟环境，每个项目有独立的 Python 和包，互不影响

## Maven vs pip 工作流对比

```text
Java (Maven):
  1. 写 pom.xml 声明依赖
  2. IDE 自动下载（或 mvn install）
  3. 直接写代码，import 即可

Python (pip):
  1. 写 requirements.txt 声明依赖        ← 手动维护
  2. 执行 pip install -r requirements.txt  ← 手动安装，不会自动触发
  3. 写代码，import 即可
```

## 实际项目中的流程

```bash
# 1. 克隆项目
git clone xxx
cd project

# 2. 创建并激活虚拟环境
python -m venv .venv
.venv\Scripts\activate

# 3. 安装依赖（这一步 Java 不需要手动做，Python 必须做）
pip install -r requirements.txt

# 4. 现在才能正常运行项目
python main.py
```

## 总结

- **Java**：pom.xml 声明 → Maven/IDE 自动安装 → 直接用
- **Python**：requirements.txt 声明 → **手动执行** pip install → 才能用

记住：拿到 Python 项目后，第一件事就是 `pip install -r requirements.txt`。
