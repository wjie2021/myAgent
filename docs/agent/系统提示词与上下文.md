# 系统提示词与上下文

> LLM 的输入由三部分组成，理解它们的区别是开发 Agent 的基础。

## 核心概念

LLM 的每次调用，传入的内容分为三层：

```text
┌─────────────────────────────────────┐
│  System Prompt（系统提示词）          │ ← 固定的，不随对话变化
│  "你是一个旅行助手，请用ReAct格式..."  │
├─────────────────────────────────────┤
│  Context（上下文/对话历史）            │ ← 随对话累积
│  用户: 查询北京天气                    │
│  AI: Thought: ... Action: ...       │
│  Observation: 晴天 31°C              │
├─────────────────────────────────────┤
│  User Input（当前用户输入）            │ ← 本次的输入
└─────────────────────────────────────┘
```

## 三个角色

| 层次 | 作用 | 是否变化 |
|------|------|----------|
| System Prompt | 定义 AI 的身份、行为规则、输出格式 | 固定不变 |
| Context | 之前的对话记录（用户输入 + AI 输出 + 工具结果） | 每轮累积 |
| User Input | 当前这一轮的输入 | 每轮新输入 |

## 实际代码示例

```python
# 系统提示词：加载一次，全程不变
with open("prompts/agent.md", "r") as f:
    AGENT_SYSTEM_PROMPT = f.read()

# 上下文：累积的对话历史
prompt_history = [f"用户请求: {user_prompt}"]

for i in range(MAX_LOOP_COUNT):
    # 上下文 = 所有历史拼起来
    full_prompt = "\n".join(prompt_history)

    # 调用时两个都传进去
    llm_output = llm.generate(full_prompt, system_prompt=AGENT_SYSTEM_PROMPT)
    #                        ↑ 上下文                ↑ 系统提示词

    # 每轮追加到上下文
    prompt_history.append(llm_output)         # AI 的输出
    prompt_history.append(observation_str)    # 工具的结果
```

## 类比理解

- **System Prompt** = 岗位职责说明书（入职时定好，不会天天改）
- **Context** = 工作日志（每天都在增长，记录所有发生过的事）
- **User Input** = 今天的任务（每次都是新的）

## 为什么要分开？

```python
# 如果不分开，每次调用都要重复"你是旅行助手"这句话
messages = [
    {"role": "user", "content": "你是旅行助手。请用Thought/Action格式回答。查询北京天气"},
    {"role": "assistant", "content": "Thought: ... Action: get_weather(...)"},
    {"role": "user", "content": "你是旅行助手。请用Thought/Action格式回答。根据天气推荐景点"},
    # ↑ 每次都要重复，浪费 token
]

# 分开后，系统提示词只写一次
system = "你是旅行助手。请用Thought/Action格式回答。"  # 写一次
messages = [
    {"role": "user", "content": "查询北京天气"},
    {"role": "assistant", "content": "Thought: ... Action: get_weather(...)"},
    {"role": "user", "content": "根据天气推荐景点"},
    # ↑ 干净，不重复
]
```

## API 调用中的体现

不同 LLM 的 API 格式略有不同，但结构一致：

```python
# OpenAI 格式
response = client.chat.completions.create(
    model="gpt-4",
    messages=[
        {"role": "system", "content": "你是旅行助手"},      # 系统提示词
        {"role": "user", "content": "查询北京天气"},         # 历史输入
        {"role": "assistant", "content": "Thought: ..."},   # 历史输出
        {"role": "user", "content": "推荐景点"},             # 当前输入
    ]
)
```

```python
# Anthropic 格式（Claude）
response = client.messages.create(
    model="claude-sonnet-4-20250514",
    system="你是旅行助手",                    # 系统提示词（单独参数）
    messages=[
        {"role": "user", "content": "查询北京天气"},
        {"role": "assistant", "content": "Thought: ..."},
        {"role": "user", "content": "推荐景点"},
    ]
)
```

## 总结

- 系统提示词 = 角色设定，固定不变
- 上下文 = 对话历史，每轮累积
- 两者独立传入，互不干扰
- 理解这个分层是开发 Agent 的基础
