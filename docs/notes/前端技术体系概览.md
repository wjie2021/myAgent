# 前端技术体系概览

> 2026-06-25，以对话方式梳理前端工具链关系，适合"会用但不知道为什么"的人。

## 1. 底层：浏览器只认识三样东西

不管你用什么框架什么工具，浏览器最终只执行：

```
HTML → 结构（页面有什么）
CSS  → 样式（长什么样）
JS   → 行为（点按钮会发生什么）
```

**所有前端工具，本质上都在做一件事：让你更舒服地写这三样，然后"翻译"成浏览器能懂的版本。**

---

## 2. JS 本身的问题 → ES6

浏览器里的 JS 曾经很弱（没有模块、没有 class 语法糖、回调地狱）。ES6（2015）加了：

- `import/export` — 终于可以拆文件了
- `const/let` — 取代 var
- 箭头函数 `() => {}` — 更短的写法
- `class` — Java 程序员看了亲切

**问题**：新语法老浏览器不认识。于是有了 Babel。

---

## 3. Babel — 翻译官

把新语法 JS（ES6+）翻译成旧浏览器也认识的 JS。

```
你的代码 → Babel → 老版本 JS → 浏览器
```

后来 Babel 还多了一个活：把 React 的 JSX 也翻译成普通 JS。

---

## 4. Webpack / Vite — 打包工

你用 `import` 拆了几十个文件，浏览器不能一个个去下载（太慢）。打包工具把这堆文件**合并成 1-2 个大文件**：

| 工具 | 特点 |
|------|------|
| **Webpack** | 老牌，React 项目（create-react-app）默认用 |
| **Vite** | 新一代，快很多，新项目基本都用 |

Babel 通常作为 Webpack/Vite 的一个"插件"嵌入 —— 打包时顺便翻译语法。

---

## 5. React / Vue — UI 框架

在 HTML/CSS/JS 之上包的**抽象层**，让你不用手写 DOM 操作：

```js
// 原生 JS：手动操作 DOM（痛苦）
document.getElementById("count").innerHTML = count;

// React：数据变了，页面自动更新
<div>{count}</div>  // 爽
```

### 三大主流

| | React | Vue | Angular |
|---|---|---|---|
| **出身** | Meta (Facebook) | 尤雨溪（个人→社区） | Google |
| **定位** | UI 库（专注视图，其他靠社区拼） | 渐进式框架 | 完整框架（全家桶） |
| **写法** | JSX/TSX（HTML 写 JS 里） | 单文件组件（template + script + style 一个文件） | TypeScript + 模板 |
| **国内市场** | 大厂主流 | 中小厂多、国内社区大 | 外企多，国内较少 |

### React/Vue 和底层的关系

> React 之于 DOM，就像 Spring Boot 之于 Servlet —— 不是"翻译"，而是"在底层之上提供一套更好用的抽象，底层脏活它帮你干了。"

无论 React 还是 Vue，最终都是在浏览器里操作同一套 DOM API。

---

## 6. Node.js — 运行环境（不是语言！）

```text
把 Chrome 的 V8 引擎拆出来，加上文件系统、网络等能力。
让 JavaScript 可以脱离浏览器，在服务器上跑。
```

### 和 Java 的对应关系

| | Java 世界 | JS 世界 |
|---|---|---|
| **语言** | Java | JavaScript |
| **运行环境** | JVM (JDK 安装包) | Node.js 安装包 |
| **运行代码的命令** | `java` | `node` |
| **包管理命令** | `mvn` / `gradle` | `npm`（装 Node.js 时附赠） |

- **语言是 JavaScript**，Node.js 只是让它能在服务器上干活的容器
- Node.js 安装包 ≈ JRE + 核心库 + npm（少了编译器，TS 编译器需另外装）

### 前端工具链全跑在 Node.js 上

Vite、Webpack、Babel、TypeScript 编译器、甚至 npm 自己，全是 Node.js 写的程序。**但最终产物（dist/ 里的文件）和 Node.js 无关，直接在浏览器里跑。**

> **Node.js 是工厂，React/Vue 是产品。产品在工厂里造，但出厂后在自己家（浏览器）用，不需要搬工厂回家。**

---

## 7. 完整流程一张图

```
你写的代码
  │
  ├── src/App.tsx        (React 组件，TSX 语法)
  ├── src/utils.ts       (普通 TS/JS)
  └── package.json       (依赖清单：React、Babel、Vite...)
  │
  ▼  npm install（下载依赖到 node_modules/）
  │
  ▼  npm run dev 或 npm run build
  │
Vite/Webpack（打包器，总管，跑在 Node.js 上）
  ├── 调用 TypeScript → 类型检查
  ├── 调用 Babel      → TSX/JSX → 普通 JS，ES6+ → ES5
  └── 把所有文件打包成 1-2 个 .js 文件
  │
  ▼
dist/ 目录（纯静态文件，和 Node.js 无关了）
  │
  ▼  丢到 Nginx / 任何静态服务器
  │
浏览器（加载 HTML + JS + CSS，跑起来）
```

- `npm run dev`：开发模式，边改边预览（类似 `java -jar` 但多了自动刷新）
- `npm run build`：出最终产物的命令
- `npm` 本身：对应的是 Maven/Gradle，不是 `java` 命令

---

## 总结

| 东西 | 是什么 | 类比 |
|------|--------|------|
| HTML/CSS/JS | 浏览器底层三件套 | JVM 字节码 |
| ES6 | JS 的新版本语法 | Java 8 → Java 17 |
| Babel | 新语法→旧语法的翻译器 | — |
| Webpack/Vite | 打包工具，合并文件 | Maven 的 assembly 插件 |
| React/Vue | UI 框架，封装 DOM 操作 | Spring Boot 封装 Servlet |
| JSX/TSX | React 里把 HTML 写 JS 里的语法 | — |
| npm | 包管理器 | Maven / Gradle |
| Node.js | JS 的服务器端运行环境 | JVM（JDK 安装包） |

**一句话：ES6/TS 是语言，Babel 是翻译官，Webpack/Vite 是打包工，npm 是采购员，Node.js 是让一切跑起来的工厂，React/Vue 是工厂生产的 UI 产品。最终产物是 dist/ 里的纯静态文件，在浏览器里跑，跟 Node.js 没关系了。**
