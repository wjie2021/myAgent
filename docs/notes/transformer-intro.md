# Transformer 入门笔记

> 2026-06-18，配合 `examples/05_transformer/` 代码学习

## 0. 前置问题：计算机怎么"读"文字？

计算机只能处理数字，所以第一步：**把每个词变成一个数字向量**（Embedding，词嵌入）。

```
"我"  →  [0.12, -0.33, 0.78, ...]   (比如 512 个数字)
"把"  →  [0.45, 0.21, -0.11, ...]
"苹果" → [-0.22, 0.67, 0.34, ...]
```

但这有个致命问题：**同一个词，不同上下文，意思完全不一样**。

---

## 1. 位置编码（Positional Encoding）—— 顺序很重要

### 问题

看这两个句子：

> "狗咬了人"  
> "人咬了狗"

词完全一样，只是顺序不同。但如果直接把词向量扔给计算机，**它看不出区别**——因为"狗"的向量还是那个向量，"人"的向量也还是那个向量。

### 解决思路

给每个词的向量**加上它的位置信息**。

```
位置:    第0位      第1位      第2位
句子:    狗         咬了        人
原向量:  [0.1,0.2]  [0.3,0.4]  [-0.1,0.5]
位置码: + [0.0,1.0] + [0.8,0.6] + [0.9,0.1]    ← 每个位置给一组独特的数字
结果:    [0.1,1.2]  [1.1,1.0]  [0.8,0.6]
```

"狗"在第 0 位时得到的向量，和"狗"在第 2 位时得到的向量——**不一样**。于是模型能区分"狗咬人"和"人咬狗"了。

**实现方式**：论文用 sin/cos 函数生成位置码（不同频率的正弦波叠加），保证每个位置有唯一编码，且两个位置之间的距离可计算。

## 2. 自注意力（Self-Attention）—— Transformer 的灵魂

### 问题

> "我把苹果吃了，**它**很甜。"

人一眼就知道"它"指"苹果"。但计算机怎么知道？

### 核心直觉

让句子里的每个词**去问其他所有词**："你跟我的关系有多大？"

每个词会产出三个向量：

| 向量 | 作用 | 类比 |
|------|------|------|
| **Q**（Query，查询） | "我在找什么？" | 你举着牌子寻人 |
| **K**（Key，键） | "我是谁？" | 每个人胸前挂的名牌 |
| **V**（Value，值） | "我的内容是什么？" | 每个人兜里揣的信息 |

### 具体过程

```
句子: "我把苹果吃了，它很甜"

"它" 想知道自己指什么：

  "它" 的 Q  ───匹配───►  "我" 的 K     → 匹配度 0.1  (不太相关)
              ───匹配───►  "苹果" 的 K   → 匹配度 0.8  (高度相关！)
              ───匹配───►  "吃了" 的 K   → 匹配度 0.3
              ───匹配───►  "很甜" 的 K   → 匹配度 0.2

然后按匹配度加权取 V：
  "它"理解的上下文 = 0.1×V_我 + 0.8×V_苹果 + 0.3×V_吃了 + 0.2×V_很甜
                        ↑          ↑
                    基本忽略    主要从这里拿信息！
```

**结果**："它"的向量被更新，包含了大量"苹果"的信息。模型现在"知道"它在说苹果。

这就是自注意力——**每个词都和所有词做一次"匹配"，从最相关的词那里拉信息过来**。

## 3. 多头注意力（Multi-Head Attention）—— 多个视角同时看

### 为什么需要"多头"？

一句话里，词之间有**不同种类**的关系：

```
"小明昨天在北京买了一本书"
```

| 头 | 可能学到的关系 | 例子 |
|----|---------------|------|
| 头1 | 主语-谓语 | "小明" ↔ "买" |
| 头2 | 时间修饰 | "昨天" ↔ "买" |
| 头3 | 地点修饰 | "在北京" ↔ "买" |
| 头4 | 宾语 | "买" ↔ "书" |

一个注意力头只能学一种关系模式。8 个头就是 8 个不同的"观察角度"，每个头关注不同的依赖关系。

**实现上**：就是把 Q、K、V 拆成 8 份，每份独立做注意力计算，最后拼回来。

---

## 4. 前馈网络（Feed-Forward）—— 增加思考能力

注意力只是**搬运信息**（从相关词那里拉信息过来）。但拉过来之后，还需要**思考处理**。

每个词的向量经过注意力更新后，再通过一个简单的两层全连接网络（线性变换 → ReLU激活 → 线性变换），让模型对该词的表示做非线性变换，增加表达能力。

---

## 5. 残差连接 + 层归一化 —— 让训练稳定

```python
x = self.norm1(x + self.dropout(attn_output))
```

- **x + attn_output**（残差连接）：原始信息 + 新信息，防止深层网络"忘掉"原始输入
- **norm1(...)**（层归一化）：把数值压到稳定范围，防止跑偏

---

## 6. 拼在一起：Encoder 和 Decoder

以翻译为例：中文 "我吃苹果" → 英文 "I eat apple"

```
输入: "我 吃 苹果"
        │
        ▼
   [词向量嵌入]
        │
        ▼
   [位置编码]        ← 加上位置信息
        │
        ▼
   ┌───────────────────┐
   │  EncoderLayer × N  │   ← 堆叠 N 层（论文里是6层）
   │  每层:             │
   │  1. 自注意力       │     "苹果"能看到"我"和"吃"
   │  2. 前馈网络       │     输出：富含上下文的词向量
   └───────────────────┘
        │
        ▼  (Encoder 的输出，理解为"读懂的语义")
        │
        ▼
   ┌───────────────────┐
   │  DecoderLayer × N  │   ← 堆叠 N 层
   │  每层:             │
   │  1. 掩码自注意力   │     生成时只能看到已生成的词
   │  2. 交叉注意力     │     看 Encoder 输出，知道原文说了什么
   │  3. 前馈网络       │
   └───────────────────┘
        │
        ▼
   输出: "I eat apple"
```

- **Encoder**：把整句中文"读透"，产出每个词在上下文中的完整语义
- **Decoder**：生成时，既看自己已经生成了什么（掩码自注意力），又看 Encoder 的理解（交叉注意力），决定下一个词该输出什么

Decoder 的"掩码"含义：生成 "eat" 时，Decoder 不能偷看后面的 "apple"——否则就是作弊。所以用 mask 把未来的词挡住。

---

## 总结：代码骨架对应关系

| 类 | 作用 |
|----|------|
| `PositionalEncoding` | 给词向量加位置信息 |
| `MultiHeadAttention` | 每个词从其他词拉相关信息（加多头视角） |
| `PositionWiseFeedForward` | 对每个词做思考变换 |
| `EncoderLayer` | 自注意力 + 前馈（理解输入） |
| `DecoderLayer` | 掩码自注意力 + 交叉注意力 + 前馈（生成输出） |

---

## Transformer 的本质

**一句话：让每个词同时看到句子里的所有其他词，然后自己决定该关注谁。**

这就是 Self-Attention（自注意力）。相比之前 RNN/LSTM 一个字一个字读、容易"遗忘"前面内容的缺点，Transformer 的并行注意力机制是革命性的突破。

现在所有大语言模型（Claude、GPT、MiMo 等）底层都是 Transformer 架构。
