用户理解 ReAct 模式后的反馈：

- 用户一直以为 ReAct 是很复杂的东西，实际理解后发现"就是类似 skill 这样的规范提示词或者输出格式的规范"
- ReAct 中的 Thought 不是模型的原生思考，而是强迫模型输出的文本
- 原生思考（如 Claude Extended Thinking、o1）是模型内部的推理过程，和 ReAct 的 Thought 完全不同

**Why:** 用户是 Java 程序员转 Python/AI，对 Agent 概念有畏惧感，需要降低学习门槛

**How to apply:** 讲解 Agent 概念时，用类比和简单语言，避免过度技术化。用户喜欢"本质是什么"的讲解方式。
