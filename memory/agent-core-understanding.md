## 用户的 Agent 理解（2026-07）

### 核心观点
- Agent 的核心是如何更好地利用 LLM
- ReAct、Plan-and-Solve 等范式本质上是"如何更好地提问"
- Agent 根据任务选择合适的范式，让 LLM 输出便于工具调用的内容
- "懒是推动进步的动力" — Agent 的出现是因为人想让 AI 自己调用工具

### 需要澄清的点（Claude 的纠正）
- **范式 vs 提示词**：ReAct 是设计范式（思想），提示词工程是实现手段，不能混为一谈
- **动机更准确的说法**：不是"人懒"，而是 LLM 有明确能力边界（无实时数据、无法执行操作），需要外挂工具扩展能力
- **Agent 的核心能力**：不只是调用工具，还包括推理、规划、决策、反思

### 用户的接受程度
- 用户理解了范式和手段的区别
- 但认为"懒"和"能力不足"本质上差不多，都是推动技术进步的原因
- 语气轻松，不纠结于用词精确性
